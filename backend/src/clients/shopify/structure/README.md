# Shopify client consolidation tracker

Working notes for centralizing every Shopify client surface in
`backend/src/clients/shopify/`. One `.md` file per fragment / query /
mutation / schema module / helper, split by source so we can tick items off as
each lands.

## Architectural target (per user's brief)

- **`.graphql` operation files + fragments** in `backend/src/clients/shopify/queries/` — ariadne-codegen generates pydantic models under `generated/`. `just code shopify` regenerates.
- **DSL-style runtime** from `shop_client.py` (schema registry: `Box(products=Resource(...), orders=..., customers=..., refunds=..., variants=...)`). Not yet folded into `shopify_client.py` — that's a Phase-2 merge.
- **Split query/variables** — never inline. Both codegen client and DSL client do this.
- **Fragments reused via subclass** — codegen produces subclass fragments naturally (`OrderCustomer(Customer)`, `ProductVariantsNodes(ProductVariant)`, `RefundTransactionsNodesAmountSet(MoneyBag)`, etc.).
- Overfetch is OK (per user); DRY via fragment reuse first, per-op tuning later if cost bites.

## Naming convention (locked 2026-07-01)

- `Get<Resource>` — direct id lookup (`GetCustomer`, `GetOrder`, `GetProduct`, `GetVariant`, `GetMediaImage`)
- `Find<Resource>s` — search/list via query string (`FindCustomers`, `FindOrders`, `FindProducts`, `FindMetaobjects`)
- `<Resource><Action>` — mutations (`CustomerUpdate`, `OrderCancel`, `RefundCreate`, `TagsUpdate`, `FileCreate`, `FileUpdate`, `FileDelete`, `InventoryAdjustQuantities`, `MetaobjectCreate`, `MetaobjectDefinitionCreate`, `ProductUpdate`, `ProductVariantsBulkUpdate`)
- One-word `<Resource>` — fragments (`Customer`, `Order`, `Product`, `ProductVariant`, `Refund`, `Metaobject`)

## Fragment hierarchy (composed → leaf)

```
Order        ──┬── Customer          (leaf)
                ├── MoneyBag ────── MoneyV2  (leaf)
                └── Refund ──── MoneyBag ← reuses

Product      ──┬── ProductVariant   (leaf)
                └── Metaobject      (leaf)  ← via importantDates.reference

Refund       ── MoneyBag ── MoneyV2

Standalone leaves:  PageInfo · ImageContent · UserError · FilesUserError · MetaobjectUserError
```
